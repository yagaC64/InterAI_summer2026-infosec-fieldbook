# Security Policy

This repo is public teaching material. It should not contain private evidence, real secrets, incident details, or exploit instructions.

## Reporting Sensitive Findings

If you find a real or suspected secret in this repo:

1. Do not open a public issue containing the secret.
2. Do not post screenshots of the secret.
3. Contact the maintainer or instructor privately with the minimum safe context.
4. Include the repo-relative path and a short description.

## Public Issues Are OK For

- broken links
- unclear instructions
- template improvements
- typo fixes
- public-safe security education improvements

## Public Issues Are Not OK For

- passwords, tokens, keys, or session values
- private identity mapping
- exploit chains
- vulnerability proof-of-concept payloads
- private repo or incident details

## Maintainer Response

Sensitive reports should be reviewed, removed from public history if needed, and followed by rotation if a real credential was exposed.
